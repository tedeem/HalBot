# HalBot: HAL 9000 AI ChatBot Simulation

[![Python Version](https://img.shields.io/badge/python-3.8%2B-blue.svg)](https://www.python.org/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**HalBot** is a local, multithreaded AI chatbot application that replicates the eerie, chillingly calm, and deeply logical persona of the HAL 9000 mainframe from *2001: A Space Odyssey*. 

Developed by Terry Deem, the project integrates local Large Language Models (LLMs) via Ollama, high-fidelity Text-to-Speech (TTS) via StyleTTS2, customized digital signal processing (DSP) filters to mimic HAL's iconic vocal timbre, and an animated retro desktop GUI.

Why?
I have always been infatuated by the HAL 9000 AI from *2001: A Space Odyssey*. It is one of the reasons I got into computers and programming. Technology like LLM's and TTS has made it possible to have my own personal HAL 9000 AI assistant. Also...I just love the Thinking Machines Supercomputer. The LED's blinking on the supercomputer is what makes it feel like a real computer. So I wanted to combine my two loves - the HAL 9000 AI and the Thinking Machines Supercomputer - into one project. But if I come back to it, I want to add voice input, recognition of the who the user is, memory of previous conversations, and maybe some personal assistant features like local RAG or web search. Who knows, maybe I'll add a camera input so the HAL 9000 can see and interact with the user's environment.... lol.

## 🚀 Key Features

* **Local AI Execution:** Uses [Ollama](https://ollama.com/) natively to ensure conversation processing stays entirely offline, private, and fast.
* **Neural Speech Synthesis:** Leverages **StyleTTS2** to generate high-quality, continuous human-like speech from text.
* **Custom Audio Engineering (DSP):**
  * **Vocal Pacing Control:** Intercepts raw audio tensors to apply a custom timeline-stretching factor using polyphase FIR filtering. This successfully forces HAL to speak with a slow, hypnotic cadence without breaking natural prosody.
  * **Studio Equalization (EQ):** Applies a high-stability Second-Order Sections (SOS) Butterworth filter matrix to cleanly isolate and boost sub-bass and chest frequencies, replicating the resonant bass response of HAL's vocal tract.
* **Retro Mainframe GUI:** Built with `Tkinter`, featuring a scrolling terminal-green chat log and a dynamic, animated LED status panel. The LEDs react to the system's state—bursting with activity during processing and fading to a dim pulse when idle.
* **Multithreaded Architecture:** Background threads handle compute-heavy LLM token streaming, TTS inference, and audio playback seamlessly so the desktop UI and LED grids stay entirely responsive and unblocked.

## 🛠️ System Architecture & Workflow

The application runs two primary processes concurrently over a centralized queue system:

1. **Main Thread (UI Loop):** Manages the Tkinter interface window, captures user keyboard input, handles auto-scrolling log inserts, and cycles the LED matrix animation loop.
2. **Background Thread (Core Loop):** Manages the Ollama conversation memory state, pipes prompts to the local model, routes text outputs through StyleTTS2, runs custom DSP algorithms, and plays the resulting audio.

**Audio Signal Processing**
Achieving the exact timbre of HAL 9000 requires more than just standard TTS. HalBot intercepts the raw audio matrix generated by StyleTTS2 and applies several transformations:

* **Pitch & Speed Manipulation:** Stretches the timeline by a precise factor (103/100) using scipy.signal.resample_poly to lower the pitch and slow the pacing.
* **Zero-Phase Filtering:** Uses sosfiltfilt across Sub, Chest, and Mid EQ bands to boost low-end frequencies while preventing harmonic smearing.
* **Edge De-clicking:** Applies a programmatic volume envelope to cleanly fade the audio in and out, preventing audio pops.

### Signal flow:
`User Input` ➔ `Ollama LLM (Text Generation)` ➔ `StyleTTS2 (Audio Synthesis)` ➔ `FIR Stretching & SOS Filter EQ` ➔ `Hardware Playback via aplay`

## 📋 Prerequisites & Requirements

HalBot is configured to run out-of-the-box on **Linux** (due to the usage of native system audio utilities). 

### System Dependencies
* **Ollama:** Core application must be installed and running in your background environment.
* **ALSA Utilities:** The system uses Linux `aplay` to play audio tensors efficiently without thread locks.

### Python Dependencies
Install the necessary processing libraries:
```bash
pip install torch numpy scipy soundfile nltk tkinter
```
(Note: You will also need to install the styletts2 package and its respective dependencies).

### Setup & Installation

1. Clone the repository:
```bash
git clone https://github.com/yourusername/HalBot.git
cd HalBot
```
2. Download the LLM:
Ensure you have pulled the required model via Ollama:
```bash
ollama run gemma4:e4b 
# OR
ollama run llama3.1:8B
```
3. Prepare the Weights & Configs:
Ensure your directory structure matches the script's expectations. You need the StyleTTS2 weights and a reference audio file of HAL 9000's voice.
- config/config.yml
- weights/hal9000.pth
- weights/hal_ref.wav

4. Run the application:
```bash
python halbot.py
```
### Usage
  1. Launch the application. You will see the HAL 9000 interface boot up and the LED panel initialize.
  2. Type your message into the input field at the bottom and press ENTER.
  3. The LED panel will burst with activity while HAL thinks and speaks.
  4. To shut down the system, type exit or quit, or use CTRL+C in the terminal to trigger the emergency override.

Note: Initial startup may take a moment as the TTS model and neural network weights are loaded into memory silently in the background.
